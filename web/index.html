WMS-SLD 1.1.0 Conformance Test Suite

Overview

A Feature Portrayal Service (FPS) is a specialized "component" WMS implementation that is able to portray feature data obtained from a remote WFS. It supports user-defined layers and styles and in general has no pre-defined layers; in this sense it is loosely coupled to any data sources. A conforming FPS implementation must satisfy the requirements of Conformance Class C in the SLD profile specification (OGC 05-078r4), plus all applicable requirements in other relevant specifications (see Figure 1).

A styled layer descriptor (SLD) is an XML resource containing definitions that associate feature types with styling rules. Its content model is described in section 11 of the SLD profile specification. Styling rules may be represented using the XML syntax defined in the Symbology Encoding specification.

What is tested

  • Feature Portrayal Service (with remote WFS: a conforming WFS 1.1 implementation)
  • Remote SLD
  • Inline SLD
  • User-defined layers
  • User-defined styles

What is not tested

  • GetLegendGraphic
  • Inline GML data
  • POST method bindings
  • Integrated WMS-SLD implementations (Clause A.2)
  • Coverage Portayal Service (Clause A.5)
